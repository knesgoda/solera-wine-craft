• Sentry SDK (@sentry/react) integrated into frontend: main.tsx init,
  ErrorBoundary.componentDidCatch → Sentry.captureException(),
  AuthContext sets user/org context tags on login.

• Verification method "envelope-accepted" means Sentry's envelope
  ingestion endpoint returned HTTP 200 (event accepted). Full API
  verification requires a SENTRY_AUTH_TOKEN with event:read scope
  (project:read is not sufficient). Upgrade token permissions to
  enable api-verified mode.

• Edge function errors are sent via manual envelope POST (no Deno
  Sentry SDK). For production, consider adding @sentry/deno when
  it reaches stable.

• The VITE_SENTRY_DSN env var must be set in the .env file for the
  frontend SDK to initialize. Currently the DSN is only configured
  as a runtime secret — add it as VITE_SENTRY_DSN in .env or as
  a build secret for the client-side integration to activate.
